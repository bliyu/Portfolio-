Capstone Project

My capstone project demonstrates collaborative software development, planning, documentation, QA support, and project organization from the early concept stage through implementation and testing.

Rogue Paddle

Rogue Paddle is a team-based desktop arcade game inspired by classic brick-breaker gameplay and enhanced with roguelike elements such as hazards, upgrades, replayability, and increasing challenge.

Tools: Lua, Love2D, GitHub, Trello, Aseprite, SQLite

My Role: QA testing, documentation support, bug fixing, deliverable organization, and helping the team maintain steady progress throughout the project lifecycle.

Project Summary

The capstone project focused on creating a desktop arcade game that is easy to understand, fun to replay, and structured around progressive gameplay, user feedback, and clear mechanics.

Focus: Overall concept, scope, and project overview

Project Vision

The vision for Rogue Paddle was to build a visually clear and engaging game experience that combines familiar arcade mechanics with replayable roguelike-style variety.

Focus: Replayability, challenge, and user experience

Project / Business Requirements

Core requirements included player control, paddle and ball interaction, score tracking, life management, game progression, hazards, upgrades, and menu navigation to support a complete gameplay experience.

Focus: Functional requirements and core features

Project Plan

The project plan was organized through team coordination, sprint goals, task assignments, milestones, and shared responsibility across design, development, testing, and documentation.

Focus: Planning, scheduling, and deliverables

Requirements Analysis and Design

Requirements analysis included identifying gameplay flow, menu structure, collision behavior, scoring logic, screen transitions, and design needs before implementation.

Focus: Planning, interface flow, and system design

Wireframes / Mockups

Wireframes and mockups were used to plan screen layout, menu navigation, gameplay structure, and the visual arrangement of key game elements before development decisions were finalized.

Focus: Layout planning and visual structure

Status Reports

Progress was tracked through regular team updates, sprint reviews, shared task management, bug tracking, and documented status reporting to support steady improvement over time.

Focus: Sprint updates, testing progress, and communication

System Implementation

The implementation stage involved building gameplay systems, testing mechanics, fixing issues, refining features, and supporting the final playable version of the project.

Focus: Development, testing, bug fixing, and refinement
